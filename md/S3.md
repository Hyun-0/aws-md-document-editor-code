# Simple Storage Service (S3)
- AWS 스토리지 서비스
- 인터넷을 통한 데이터 접근
- 백업, 아카이브, 대규모 데이터 저장

### S3의 주요 기능
- 무한 확장 가능성
  - 저장할 수 있는 데이터의 크기 제한 없음

- 고가용성
  - 여러 리전에 걸쳐 데이터 복제
  - 서비스 가용성 높임

- 데이터 내구성
  - 99.999...%의 데이터 손실 없음
  - 데이터를 장기간 안전하게 보관 가능

- 보안 기능
  - 암호화
  - IAM을 사용하여 사용자 권한 관리
  - 버킷 레벨에서 정책을 설정하여 보안 강화 

### S3 저장 구조
- 버킷 (Bucket)
  - S3에서 데이터를 저장하는 기본 단위

- 객체 (Object)
  - S3에 저장되는 데이터 단위

- 키 (Key)
  - 각 객체를 식별하는 고유 문자열

### S3 데이터 접근 방식
- REST API
  - HTTP 요청을 통해 데이터 저장, 검색, 삭제 가능

- Console
  - 웹 인터페이스로 S3 데이터 시각적 관리

- SDK
  - 다양한 프로그래밍 언어로 S3 접근

### S3 비용 구조
- 사용량 기반 비용
- 저장 계층에 따른 비용
- 데이터의 접근 빈도에 따라 저장 계층을 변경해 비용 절감 가능

### S3 활용 사례
- 정적 웹 애플리케이션 호스팅
- 빅데이터 분석
- 백업 및 복구

### S3와 다른 AWS 서비스의 연동
- Lambda, CloudFront, EC2 등과 쉽게 통합 가능
- 데이터 처리, 전송, 웹 배포 지원
- Amazon Athena와 통합하여 S3에 저장된 데이터를 SQL 쿼리로 분석 가능 

### S3의 장점
- 무한한 확장성
- 내구성
- 가용성
- 비용 효율성
